Principles that turn chaos into craft

State > Compute

Durable beads outlive any worker. Every decision is anchored in the ledger, not in ephemeral memory.

Spec-driven output

Humans define the schema. Agents deliver only implementation — no architecture drift, no surprises.

Nondeterministic, idempotent

Each attempt can differ, but the artifact converges to the same verified result.

Continuous integration as law

No merge without passing gates. The refinery rejects anything that fails tests, lint, or spec.

The Foundry Architecture

The Workflow Engine

Compiles static TOML Formulas into dynamic Molecules (task graphs), ensuring strictly reproducible processes.

The Router

Orchestrates the "Foundry". Analyzes Formulas, instantiates Beads, and manages the dependency graph (DAG).

The Worker (Dynamic)

Stateless executors that can recursively delegate (Dynamic Bonding) when tasks exceed single-agent complexity.

Project Awareness

Agents automatically discover and adhere to AGENTS.md rules, adapting to your specific repo constraints.

Resilient Recovery

Automated on_failure logic ensures reliability. The system skips recovery steps on success, but executes them immediately on failure.

The Citadel pipeline

1. Intake Requests or Formulas submitted to Router
2. Cooking Workflow Engine compiles Formulas to Molecules
3. Execution Workers resolve Beads and delegate sub-tasks
4. Verification Gatekeepers validate work or trigger Recovery steps on failure
5. Delivery Verified Molecules merged to Convoy
